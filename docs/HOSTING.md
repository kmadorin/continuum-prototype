# Hosting Continuum (the "live product" link)

The whole product is ONE deployable: the custody backend (`app/custody/server.ts`) serves both the JSON API
and the built React SPA (`app/web/dist`). Deploy it anywhere that runs a Node container (Render / Fly / Railway).

## Secrets to inject at deploy (NEVER commit these)
| Env var | What |
|---|---|
| `FN_SECRET` | 5N devnet M2M client secret (transport token). From the "Seaport Sandbox Validator Access" PDF (the `…<REDACTED-SECRET-FRAGMENT>…` value — use `pdftotext`, one char is a capital I). |
| `CUSTODY_KEYS_JSON` | The contents of the gitignored `app/custody-keys.json` (the 5 per-party mnemonics), pasted as one JSON string. Generated by `npx tsx custody/provision.ts`. These are throwaway devnet demo wallets. |
| `CUSTODY_SESSION_SECRET` | Any long random string (signs session cookies). |
| `PORT` | Optional (default 8787; set to the platform's expected port). |

## Provision the 5 demo wallets (once)
```
cd app && npx tsx custody/provision.ts     # writes gitignored custody-keys.json + party-registry.json
cat custody-keys.json                        # → paste into the CUSTODY_KEYS_JSON secret on the host
```

## Deploy (Docker; e.g. Render/Fly)
```
# from app/
docker build -t continuum .
docker run -p 8787:8787 \
  -e FN_SECRET="…" -e CUSTODY_KEYS_JSON="$(cat custody-keys.json)" \
  -e CUSTODY_SESSION_SECRET="$(openssl rand -hex 32)" continuum
# → open http://localhost:8787
```
On Render/Fly: point at `app/Dockerfile`, set the 4 env vars as secrets, deploy. The resulting URL is the
hackathon "live product" link.

## Before publishing the repo PUBLIC
- `gitleaks detect --source .` → no findings.
- Confirm `custody-keys.json`, `wallet-keys.json`, `.env` are all gitignored (they are).
- Seed the public repo from the scrubbed tree (see the Stream-C plan) or push `integration` after a gitleaks pass.
- Rotate the 5N M2M secret after the event.
