## 1. Local sandbox — the whole demo without a devnet (`5d97bc9`, `e4d647c`)

**Why.** The devnet party keys were unavailable, and frontend work needed a fast, safe loop.

**What.** The real Hono `createApp(deps)` backend is booted inside the Vite dev server with three ports swapped: the signer, the fetch implementation, and the read path all point at an in-memory `FakeLedger`. The fake is not a stub — it reproduces the properties the demo exists to show:

- **Privacy projection**: `projectionOf()` computes each party's view from the same signatory/observer rules as the Daml templates, so a seat literally cannot read a contract the ledger would not hand it.
- **Choice authorization and consumption**: controllers are enforced; consuming choices consume.
- **Atomic Close**: the full leg set (ValidateIssuance, transfers, burns, disclosures) executes against a snapshot with a unit-conservation assert — any failure rolls the whole transaction back.

Tenant keys are **deterministic** (`sha256('continuum-sandbox-' + role)` → mnemonic), so party ids are identical across dev-server restarts and a signed-in browser tab keeps working. This fixed the "metrics never load" bug class where a stale session cookie pointed at a party that no longer existed.

## 2. Audit, then correctness (`5535311`, `9f69424`, `95c6b8d`, `6203ba6`)

A full click-through audit of all six seats is recorded in `docs/ui-audit-2026-07-14.md`, and the fixes landed in the order the user set: money first, oversight second, copy/interaction last.

- **The cap table composes** (`9f69424`). The demo economics are now one consistent story: reference NAV $500M = $300M exiting + $200M rolling; at the 96% clearing price the buyer funds $288M cash and receives 288M CV units, rollers receive 192M units, and `288M + 192M = 480M = psaPrice` — the exact conservation the on-ledger Close asserts. Every seat's screen shows numbers that add up to what actually settles.
- **Oversight without leaking** (`95c6b8d`). The GP is not a stakeholder of a `SealedBid` or an `LPElection`; a page that polls for them counts zero forever. Oversight now runs on the **contentless markers** (`BidFiled`, `ElectionFiled`) that carry no amounts — the GP reads "2 of 2 responded" with zero private contracts in its projection, and tests pin that the page never even asks the ledger for a template it cannot see.
- **Copy stops contradicting the ledger** (`6203ba6`). The Valuer seat no longer claims an audience its `observer gp` projection does not give it; stage hints, gate copy, and post-close states were aligned with what is actually on-ledger.

**Deferred by decision** (recorded in the audit doc): LPAC valuation visibility (a one-line Daml observer change + redeploy) and the deal-naming question (Meridian CV I vs Project Continuum CV I, L.P.). Two cosmetics remain open: the activity feed lacks a close entry, and the `LPACConsentRequest → Grant` path is unreachable in the current flow.

## 3. Institutional redesign (`85bad3b` and the polish train)

A ground-up visual system, applied product-wide with all functionality, terminology, roles, permissions, and statuses preserved:

- **Tokens.** Five dark neutral surfaces, thin borders (dimmed twice on request — final `#232B36 / #2F3947 / #1B212A`), one indigo action color `#4F67E8` with an indigo→violet support ramp, and a strict semantic rule: **green appears only on confirmed on-ledger facts** (settled, anchored, recorded) — never decoratively.
- **Type.** Signifier (rare, editorial — page titles, the deal name) over Söhne (all functional UI), loaded as Newsreader + Geist. **No monospace anywhere**: data alignment comes from tabular numerals, not a second family.
- **App shell.** Persistent sidebar (brand + deal context + section nav as a real WAI-ARIA vertical tablist + signing identity/custodian footer), sticky translucent page header, full-width content. The deal-status slot in the sidebar never renders empty: skeleton pill → stage chip → a quiet dash when the seat's projection has no deal.
- **Components.** Cards/panels, chips whose workflow states are distinguishable beyond color (dot + label), KPI tiles with as-of lines, key-value tables, a bounded `SliderField` (big slider target + exact numeric entry + min/max caps — replacing tiny spinner buttons), redaction bars for sealed values, and the "cannot see" privacy note as a product feature, not an apology.

## 4. Sign-in as the product's face (`ce58a0f`, `8025692`, and this batch)

- Brand panel: eyebrow, serif headline, one-sentence description with a deliberate line break, five USP bullets, CTAs ("Enter demo workspace", "See how it works" → the internal `/pitch-deck` route that recreates the deck in-repo).
- **Flow field**: barely-there interweaving SVG streams; a brighter copy of the same lines sits under a **cursor-following radial mask**, so the streams glow where the pointer travels (rAF-eased attribute writes, zero React re-renders; lights up fast, fades out slowly when the cursor leaves).
- **Aurora**: two oversized layers of pre-softened radial glows in the brand palette drifting on `transform` only — compositor-friendly, no filter animation, disabled under `prefers-reduced-motion`.
- Account list: 48px avatars, hover-revealed CTAs, whole-card hover in the open state, and a per-role **"open in a new window"** link (`/?seat=<role>` deep link lands with that seat's login panel ready) — the demo is meant to be watched from several seats at once.
- Footer: demo note and custodian note merged into one two-line paragraph, Reset demo bottom-right, bottoms aligned with the brand panel's small print.

## 5. Motion system (emil-design-eng principles)

- **Entrances are pure dissolves** — no vertical travel anywhere (`rise-in` → `fade-in`, toasts included). Staggered per-section delays.
- **Asymmetry with intent**: the inspector drawer enters slowly (500ms iOS-drawer curve) and leaves fast (200ms strong ease-out), strictly right→left, no fade, no shadow, backdrop lands instantly — the slide stays on the compositor. Sign-out dissolves the whole workspace (260ms) before the session drops, so the swap to sign-in reads as a crossfade.
- **Toasts**: a plain column, newest at the bottom; smooth fade-in, longer lifetimes (9s success / 14s error), a draining countdown hairline per toast, hover pauses both the JS timer and the CSS bar in lockstep, and dismissal is a fade, not a pop.
- **Loading**: native-feeling preloaders (breathing dash in real layout, not gray skeleton blocks), a 500ms minimum display floor so first paint doesn't pop, and a one-time dissolve when figures land.

## 6. Mobile and responsive

- **≤960px the sidebar dissolves into an app frame**: one fixed translucent header row (logo + horizontally scrollable section tabs, backdrop blur) and the identity/custodian footer at the very foot of the page. `display: contents` + flex order does the re-arrangement without duplicating DOM.
- Scroll resets on entering the shell and on section change — previously the sign-in accordion's `scrollIntoView` survived the SPA swap and the workspace opened 550px down.
- Ancillary fixes: `scrollbar-gutter: stable` (no sideways shell jump between pages), stacked page header under 1240px with the stepper in its own scroll lane, 2×1 KPI grid, single-column forms/kv tables, safe-area padding on the mobile footer.

## 7. Anchored documents in the house style (`7a482d1`)

The three served documents (`valuation-report`, `fairness-opinion`, `purchase-agreement`) were paper-white letters with three different accent colors. They now share the workspace's exact tokens and type — dark panel on dark ground, serif titles, pale uppercase section labels, primary-accent opinion callouts and NAV box, tabular numerals — with **content untouched**. Because the ledger anchors the **sha256 of the served bytes**, `manifest.json` and `hashes.ts` were regenerated in the same commit, keeping `GET /verify/:name` honest. (A running sandbox needs a restart + Reset demo to re-anchor with the new hashes.)
